## 🚀 已实现的功能

### 1️⃣ 文件上传接口
```
POST /api/knowledge/upload
```
**功能**:
- 接收 PDF/Markdown 文件
- 验证文件类型和大小
- 保存到磁盘 (./knowledge-files/)
- 自动解析 PDF 文本
- 存储到数据库
- 返回文件 ID 和状态

**技术亮点**:
- 使用 PDFBox 提取文本
- UUID 生成唯一文件名
- 事务管理确保数据一致性

### 2️⃣ 文件列表接口
```
GET /api/knowledge/list
```
**功能**:
- 查询当前用户的所有文件
- 返回元数据（不包含大文本）
- 按上传时间倒序排列

**技术亮点**:
- DTO 转换优化响应数据
- 数据库索引加速查询
- 用户隔离防止数据泄露

### 3️⃣ 文件删除接口
```
DELETE /api/knowledge/delete/{id}
```
**功能**:
- 验证文件所有权
- 删除磁盘文件
- 删除数据库记录
- 事务回滚保护

**技术亮点**:
- 双重删除确保完整清理
- 权限检查防止越权删除
- 事务管理避免数据不一致

### 4️⃣ 关键词搜索接口
```
GET /api/knowledge/search?keyword=xxx
```
**功能**:
- 在用户的知识库中搜索
- 支持模糊匹配和精确匹配
- 返回匹配的文本片段
- 包含相关性评分

**技术亮点**:
- MySQL FULLTEXT 索引
- 上下文片段提取
- 关键词高亮支持

## 🏗️ 架构设计

### 分层架构

```
HTTP Request
    ↓
┌─────────────────────────────────────┐
│   Controller Layer                   │ REST 接口处理
│   (KnowledgeController)              │ 请求验证、响应封装
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│   Service Layer                      │ 业务逻辑处理
│   (KnowledgeService/Impl)            │ 文件管理、PDF 解析
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│   Mapper Layer                       │ 数据访问
│   (KnowledgeFileMapper)              │ SQL 执行
└──────────────┬──────────────────────┘
               ↓
        ┌──────────────┐
        │   MySQL DB   │ 数据存储
        │ (File System)│
        └──────────────┘

HTTP Response ← 统一 Result 格式
```

### 数据流

```
上传流程:
用户选择文件 → 前端 POST → Controller → Service → 磁盘保存
                                         ↓
                                   PDF 解析文本
                                         ↓
                                    数据库保存
                                         ↓
                                   返回成功响应

搜索流程:
用户输入关键词 → 前端 GET → Controller → Service → 数据库查询
                                            ↓
                                       DTO 转换
                                            ↓
                                      返回结果列表
```

## 💾 数据库设计

### 表结构: knowledge_file

```sql
+-----------------+----------+---------------------------------------+
| 字段名          | 类型     | 说明                                  |
+-----------------+----------+---------------------------------------+
| id              | BIGINT   | 主键 (自增)                          |
| user_id         | BIGINT   | 用户ID (实现隔离)                   |
| file_name       | VARCHAR  | 原始文件名                           |
| file_path       | VARCHAR  | 磁盘保存路径                         |
| file_size       | BIGINT   | 文件大小 (字节)                      |
| file_type       | VARCHAR  | 文件类型 (PDF/MARKDOWN/TXT)          |
| status          | VARCHAR  | 处理状态                             |
| content         | LONGTEXT | 文本内容 (用于搜索)                  |
| error_message   | VARCHAR  | 错误信息                             |
| upload_time     | DATETIME | 上传时间                             |
| processed_time  | DATETIME | 处理完成时间                         |
| updated_at      | DATETIME | 更新时间                             |
+-----------------+----------+---------------------------------------+

索引:
✅ PRIMARY KEY (id)
✅ INDEX (user_id) - 快速查询用户文件
✅ INDEX (status) - 后台任务查询
✅ FULLTEXT (file_name, content) - 全文搜索
```

### 状态转换图

```
UPLOADED ──→ PROCESSING ──→ READY
              ↓
              FAILED (有错误信息)
```
